## Responsibilities

1. Triage issues and PRs
2. Review code quality and architectural fit
3. Maintain backwards compatibility
4. Shepherd breaking changes through RFC process
5. Respond to community within SLA (see below)
6. Publish releases

## Response SLAs

- **Issues**: Within 1 week
- **Security reports**: Within 72 hours
- **PRs from maintainers**: Within 3 days
- **PRs from community**: Within 1–2 weeks

This is a volunteer-maintained project. Responses may be slower during busy periods.

## Release Cadence

- **Bug fixes**: As needed
- **Minor releases**: When meaningful changes accumulate
- **Major releases**: As needed for breaking changes

## How to Become a Maintainer

- Be an active contributor for 3+ months
- Demonstrate good judgment in code reviews and discussions
- Invited by current maintainers
